The Gardener's Planting System

by Yvonne V. Richardson

The GPS web site provides information to the average gardener about plants that they might be interested in growing.

On the Home page, the user selects from the nav bar, and a column of information is displayed. When they select an item from that column, it is highlighted, and then they can choose 1 of 3 pages based on that column selection. There is a button that is displayed when they make their selection, or they can choose from the pages in the nav bar.

For example, I can choose flowers from the first page, and then select "petunia" from the list. From there I can ask what petunias look like(life stages), what to plant them with (companions), or just more stuff about petunias("more about" button).

The web page is responsive, but could be fine-tuned a little more to reduce jumpiness as the user changes screen sizes. The footer that always stays on the page occasionally obscures some text, which is a reminder that they may need to scroll to read the content (in addition to respecting the copyright).

The Gardener's Planting System is above and beyond what we are required to do for our final project. Most of the pages are populated from data in several files; there are about 50 entries in the files although some entries contain less information than others. A later edition would use MySQL instead of files in the files folder. Some of the file entries are not completely filled out; for example, if there are no additional comments on a file entry, then the Comment field is blank.

Because the Website uses Javascript and PHP in addition to HTML and CSS, some pages are generated in addition to the static pages in the root directory. These languages are used to pass cookies between pages, so that the user can select an item on the first page and get detailed information of various types on the other pages. If they navigate to a page without selecting a plant first, the site displays a reminder that they should select a plant.

The class requirements are 5 pages minimum, therefore some of the HTML pages may have included Lorem Ipsum text. Each page uses main.css for common styling, and also has its own .css file for styling that was added to the project for that page. Much of the responsive programming is based on screen percentages, although there are media queries that are specific to each page.

The web site uses a consistent navigation scheme by using the main navigation bar on every page, including the home page. In addition, navigation on the home page is used to select from the various files that contain information. No default fonts are used in the site, and there are alt tags for all images.

The only default images in the site are the ones that are associated with the contents of the files. At the moment there are nearly 50 entries in the Gardeners Planting System, which means about 400 images of plants when the database is complete. Oranges are used for "under construction" images until all of the data can be collected.

Meta Tags and Accessibility

Meta tags for keywords and descriptions are added to Web pages as much as they used to, because search engines are so much more adept at finding web sites. I added some to make the page easier to find, should I ever turn it into a real Web site. There are also alt-tags for every image.

Design Principles

I rely on the design principles of Proximity, Alignment, Repetition, and Contrast to make the page flow together, regardless of size.

One factor in deciding the overall design was that the web-page will be content-heavy, therefore the images and the responsiveness should not overshadow the text that is the reason for the site's existence.

Another factor is to allow for expansion. The number of pages will always be the same, but the amount of data in the files will change. The Web page is file-based, so images of oranges are used to indicate when a file entry is in process.

A third factor is reasonableness in responsiveness. People are still accustomed to web pages that may not quite fit the device they have. When items do not display perfectly, the average user will still consider them to be reasonably readable.


Other principles used in creating the web page are as follows.

  • Balance

    The pages are nicely balanced, and the text flows in to fill in the space. The pages that display file information repeat the same two-in-a-box format that is easy to populate from the javascript and php behind the pages. The home page sequence is a little unbalanced when it displays a list from which to select, but the box is the same size as the pages that occur before and after, so the list format is not as noticeable. The alignment depends on whether there is text to display, so some of it aligns better at some screen sizes than others.

  • Grid

    I used grids to line up content on secondary pages. I did not use a spreadsheet-type grid because the web site does not display enough information on any page to make good use of the grid.

  • Color

    The colors for backgrounds and text were sampled from the image that is in the main header and on the contact page. This adds to the sense of harmony to the site.

  • Graphics

    I kept graphics to a minimum so that users would concentrate on language content, or images of the plants.

  • Typography

    I kept the number of fonts to a minimum and selected some favorite serif fonts that are somewhat reminiscent of the Farmers Almanac. The contrast is good without being glaring.

  • White space

    The backgrounds for language content are white for easy reading. The areas are properly padded to give the make the text stand out and invite the user to read a little longer.

  • Connection

    The user should feel that they can read as long as they need to, and they will probably experiment with the Web page design a little.
